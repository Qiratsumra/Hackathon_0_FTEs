## Purpose
This skill allows the AI Employee to retrieve recent posts or media from a specified social media platform (Facebook Page, Instagram Business Account, or Twitter User Timeline). The retrieved content can then be processed by the AI's reasoning engine (Claude) for tasks like summarization, sentiment analysis, or trend identification.

## Input Parameters
The AI Employee needs the following information to retrieve social media posts:

1.  **`platform` (string, required):** The social media platform to retrieve posts from. Valid values are: "facebook", "instagram", "twitter".
2.  **`identifier` (string, required):** The identifier for the target on the platform.
    *   For "facebook": The Facebook Page ID.
    *   For "instagram": The Instagram Business Account ID.
    *   For "twitter": The Twitter username (e.g., "Google").
3.  **`limit` (number, optional):** The maximum number of recent posts/media items to retrieve. Defaults to 5.

## Output
Upon successful retrieval of social media content, the skill will return:
-   `success` (boolean): `true` if content was retrieved, `false` otherwise.
-   `platform` (string): The platform from which content was retrieved.
-   `identifier` (string): The identifier used to retrieve content.
-   `posts` (list of objects): A list of dictionaries, each representing a post or media item. The structure of these objects will vary slightly by platform but will generally include: `id`, `text`/`caption`/`message`, `created_at`/`timestamp`, and `permalink_url`/`media_url`.
-   `message` (string): A descriptive message about the outcome.

If the skill fails, it will return:
-   `success` (boolean): `false`.
-   `message` (string): An error message explaining the failure (e.g., "Platform not supported", "Invalid identifier", "Authentication error", "API rate limit exceeded").

## Execution Steps (Internal AI Logic)
1.  **Validate Inputs:** Ensure `platform` and `identifier` are provided.
2.  **Invoke Appropriate MCP:**
    *   If `platform` is "facebook": Call `facebook-instagram-mcp.get_facebook_page_posts` with `identifier` (as `page_id`) and `limit`.
    *   If `platform` is "instagram": Call `facebook-instagram-mcp.get_instagram_media` with `identifier` (as `ig_user_id`) and `limit`.
    *   If `platform` is "twitter": Call `twitter-mcp.get_user_tweets` with `identifier` (as `username`) and `limit`.
    *   If `platform` is not supported: Return failure.
3.  **Handle Response:** Process the response from the MCP, ensure data is structured consistently if possible, or pass raw.
4.  **Report Result:** Log the outcome and present the retrieved `posts` if successful.

**User Request:** "Get the 3 most recent posts from our company's Facebook page to summarize them."

## Security & Approval Considerations
-   Retrieving public social media content generally does not require human approval, but the subsequent use or sharing of that content might.
-   Ensure API credentials for all platforms are securely managed.
-   All social media interactions are logged by their respective MCPs.
-   The AI should respect platform terms of service regarding data usage and privacy.
